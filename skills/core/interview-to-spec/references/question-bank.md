# Interview Question Bank

## Discovery Phase

### Problem Understanding
- What triggered the need for this?
- How are you solving this today?
- What's painful about the current solution?
- How much time/money does this problem cost?
- Who else has this problem?

### User Understanding
- Describe your typical user in detail
- What's their technical sophistication?
- What devices do they use?
- When/where do they need this?
- What other tools do they use alongside this?

### Success Metrics
- How will you measure success?
- What numbers would make this a win?
- What's the timeline for seeing results?
- What's the minimum viable outcome?

## Requirements Phase

### Functional Requirements
- What are the core actions users need to take?
- What information do users need to see?
- What decisions do users need to make?
- What's the happy path flow?
- What variations of the flow exist?

### Data Requirements
- What data needs to be captured?
- What data needs to be displayed?
- Where does the data come from?
- How often does it change?
- How long should it be retained?

### Integration Requirements
- What systems need to connect?
- What direction does data flow?
- Real-time or batch processing?
- What's the fallback if integration fails?

## Edge Cases

### Empty States
- What does the user see with no data?
- How do they get started?
- What's the onboarding flow?

### Error States
- What can go wrong?
- How should errors be communicated?
- What recovery options exist?

### Scale Extremes
- What if there's 1 item? 1 million items?
- What if the name is 1 character? 1000 characters?
- What if 100 users do this simultaneously?

### Permission Scenarios
- Who can view this?
- Who can edit this?
- Who can delete this?
- What happens when permissions change?

## Prioritization

### MoSCoW Questions
- "If we could only ship ONE thing, what would it be?"
- "What can we defer to v2?"
- "What would you cut if we had half the time?"
- "What's a dealbreaker if missing?"

### Effort vs Impact
- "Which feature gives 80% of the value?"
- "What's surprisingly complex?"
- "What's surprisingly simple?"

## Technical Constraints

### Performance
- What response time is acceptable?
- What's the expected load?
- Peak usage patterns?

### Security
- What data is sensitive?
- Compliance requirements (GDPR, HIPAA, SOC2)?
- Authentication requirements?

### Infrastructure
- Cloud provider preferences?
- Existing tech stack constraints?
- Budget constraints?

## Closing Questions

- What haven't I asked that I should have?
- What keeps you up at night about this project?
- What would make you regret building this?
- If this is wildly successful, what problems does that create?
